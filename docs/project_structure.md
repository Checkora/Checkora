# Project Structure

This document explains the directory layout of Checkora.

## Root Structure

```text
Checkora/
├── .github/
├── api/
├── core/
├── docs/
├── game/
├── README.md
├── CONTRIBUTING.md
└── requirements.txt
```

## Directory Overview

### `.github/`
Contains GitHub Actions workflows, issue templates, and repository automation.

### `api/`
API routes and backend request handling logic.

### `core/`
Core application logic and business functionality.

### `docs/`
Project documentation, guides, architecture notes, and troubleshooting resources.

### `game/`
Frontend/game-related source files.

### `README.md`
Main project overview, installation guide, and contributor entry point.

### `CONTRIBUTING.md`
Contribution workflow, coding standards, and pull request guidelines.

### `requirements.txt`
Python project dependencies.
